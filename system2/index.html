System2

A framework for deliberate, spec-driven, verification-first software engineering. Instead of ad-hoc prompting, an orchestrator coordinates specialized subagents through explicit quality gates—from scope to ship.

Install Repository

Workflow

Eight stages, six approval gates

Progress is deliberate: each planning artifact is reviewed before the next stage unlocks. Implementation and verification run only after tasks are approved.

  1. Gate 0 Scope Goal, constraints, definition of done
  2. Gate 1 Context Approve spec/context.md
  3. Gate 2 Requirements EARS-format spec/requirements.md
  4. Gate 3 Design Architecture in spec/design.md
  5. Gate 4 Tasks Atomic work in spec/tasks.md
  6. Execute Implementation Small diffs via executor agent
  7. Verify Verification Tests, security, evals
  8. Gate 5 Ship Final diff and risk checklist

Capabilities

What you get

13 specialized agents

Purpose-built subagents for context, requirements, design, tasks, implementation, testing, security, evals, docs, review, postmortems, and MCP tooling.

Safety hooks

Validation scripts guard dangerous commands, sensitive files, path boundaries, and change budgets before work lands in your repo.

Per-agent allowlists

Regex-based file restrictions keep each agent in its lane—planners don’t rewrite production code; executors don’t skip verification.

Overlay-ready

Extend System2 with overlays that inject guidance into the agent pipeline without forking the core plugin. Browse the overlay library.

Install

Add System2 to Claude Code

Install via the Claude Code plugin marketplace, restart to load agents and hooks, then initialize orchestrator instructions in your project.

Quick start

curl -fsSL https://deliberatecode.github.io/system2/install.sh | bash

Or follow the manual steps below.

Step 1 — Add marketplace

/plugin marketplace add DeliberateCode/System2

Step 2 — Install plugin

/plugin install system2@system2-marketplace

Step 3 — Restart Claude Code

Restart Claude Code so the new agents, hooks, and skills load.

Step 4 — Initialize

/system2:init

Step 5 — Restart Claude Code

Restart again so the new CLAUDE.md orchestrator instructions take effect.

Step 2 installs all 13 agents, hooks, allowlists, and the /system2:init, /system2:compose, and /system2:doctor skills. Run /system2:init in your project root to write orchestrator instructions to CLAUDE.md; use /system2:init --force to replace an existing file. Check plugin status with /plugin list. After plugin updates, run /system2:doctor and recompose overlays with /system2:compose --from-lock if needed.
