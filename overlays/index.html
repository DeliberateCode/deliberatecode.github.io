Reusable composition

Overlay profiles

A profile is a named, reusable set of overlays. Settle on a useful combination once, then activate it by name in any project instead of retyping overlay paths. Profiles live at ~/.system2/profiles.json—user-level, shared across every project on your machine, and independent of each project’s .system2/overlays.json. 

# Activate a profile in the current project (dry-run preview first)
/system2:compose --profile backend-stack

# Save the project's current overlay set as a profile
/system2:compose --save-profile backend-stack

# Define a profile explicitly from overlay paths
/system2:compose create backend-stack /path/to/overlay-a /path/to/overlay-b

# Adjust a profile (flags are repeatable)
/system2:compose edit backend-stack --add /path/to/overlay-c --remove OverlayA

# Inspect, read-only — list profiles or show one profile's set
/system2:profile list
/system2:profile list --profile backend-stack

The /system2:compose namespace creates, activates, and changes profiles; /system2:profile is strictly read-only. Activation is all-or-nothing—if a profile references an overlay path that is missing or no longer valid, it fails cleanly, names the offending path, and writes nothing.
